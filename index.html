Phabricator

Phabricator

http://phab.core.kran


Phabricator - это комплексная opensource система для ведения IT проектов родом из Facebook

Он включает в себя большое количество плагинов, постоянно развивается, используется в ряде очень крупных IT компаний: Facebook, Discuss, Dropbox, Haskell, DeviantArt и др.

Phabricator

В центре системы лежит сущность Проекта, с ней тесно интегрированы все остальные плагины и подсистемы.

Phabricator

Итак, функциональность, которая может нам пригодиться:

  1. Maniphest - трекер задач и ошибок
  2. Diffusion - хостинг Mercurial, Git, SVN репозиториев
  3. Differential и Audit - code-review
  4. Paste - сниппеты
  5. Phriction - Wiki
  6. API

Projects & Maniphest

  1. Трекинг задач и ошибок в рамках проектов
  2. Подписка на обновления по проекту или по задаче. Очень гибкая и удобная.
  3. Создание задачи через отправку email на адрес task@phab.core.kran
  4. Kanban доски, Scrum, спринты
  5. Привязка репозиториев к проектам
  6. Вся информация - в одном месте
  7. Очень много менее значительных "плюшек"

Kanban? Scrum?

Многим понравился принцип работы в Trello, но не нравилось что критически важная информация хранится на "вражеских" серверах в облаке. Phabricator живет на наших собственных серверах и закрыт извне (на самом деле нет).

Diffusion - хостинг исходного кода

Функциональные особенности

  1. Хостинг Mercurial, Git, SVN репозиториев
  2. Клонирование по HTTP и SSH (to-do)
  3. Визуализация процесса ветвления
  4. Возможность развить идею до CI (сборка, тесты, Jenkins, деплой)
  5. Возможность привязки коммитов к задачам (to-do)
  6. Code-review

Code-review и Phabricator

Code-review работает даже в микрокомандах от 2-ух человек.

  1. Коммит в репозиторий проекта
  2. Назначение ревьюеров (аудиторов)
  3. Ревью кода. Approve или decline коммита.

Весь цикл сопровождается уведомлениями по почте и в ленте активности

Code-review и Phabricator

Преимущества

  1. Стыдно писать плохой код
  2. Выравнивается "скилл" команды
  3. Все заинтересованные в курсе происходящего внутри проекта
  4. Уходит проблема "замыленного" глаза
  5. Более быстрое обучение новичков

Трудно понять преимущества code-review не попробовав.

Paste - сниппеты кода

  1. Миграция кода из проекта в проект
  2. Реализация рутинных задач
  3. Шаблоны конфигураций

Phriction - wiki

  1. Документация по проектам
  2. Справочная информация по инфраструктуре (например, доступ к серверам)
  3. Другая корпоративная информация

Недостатки решения

  1. Интерфейс - на английском (есть пакеты для интернационализации)
  2. К интерфейсу нужно привыкнуть
  3. Нет некоторых фич, которые есть в платных аналогах (продукты Atlassian)
  4. Не очень прозрачная работа с подзадачами

Аналоги

  1. Продукты Atlassian - JIRA + Stash + HipChat + Confluence. Очень крутые, но платные и достаточно дорогие
  2. YouTrack от JetBrains. Платный
  3. Отдельные системы - Redmine, SCM Manager и др. Интеграция между такими системами достаточно кривая

Sentry

Централизованный мониторинг ошибок

http://sentry.krasinform.ru

  1. Все ошибки - в одном месте
  2. Асинхронность и real-time мониторинг
  3. Очень простая интеграция в готовый проект
  4. Оповещение об ошибке по почте
  5. Исчерпывающая информация по ошибкам
  6. Логгировать можно все что угодно и это не потребует больших усилий

Простая интеграция - C#

  1. Установить raven-csharp (есть на github)
  2. Инцициализировать клиент 
                        ravenClient = new RavenClient("...");
